How to connect Google Workspace token (guide)
  1. Open Google Cloud project creator, create/select a project, then open API Library and enable APIs you need (for example Google Meet + Workspace Events).
  2. Configure the OAuth consent screen (usually Internal for Workspace organizations), then add required scopes.
  3. Go to Credentials → Create credentials → OAuth client ID. Save your Client ID and Client secret, and set your backend callback URL in Authorized redirect URIs.
  4. Run OAuth once to get tokens:
    1. Use your backend auth route or open the OAuth 2.0 Playground.
    2. In Playground: click the gear icon, check Use your own OAuth credentials, and paste your client ID/secret.
    3. Pick required scopes, then click Authorize APIs.
    4. Click Exchange authorization code for tokens.
    5. Copy the refresh token (long-lived) to your backend secret store. Do not paste refresh token into this page.
  5. Your backend uses refresh token to mint access tokens and call Google APIs, then expose a safe feed endpoint for this page (the Meet event feed URL field).
  6. For this screen's Event feed token field, create your own shared token in backend (for example a random 32+ character secret) and send it as Authorization: Bearer YOUR_TOKEN. This token is for your backend endpoint, not a raw Google OAuth token.
